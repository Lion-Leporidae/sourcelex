# 🧠 RepoMind 架构思维导图

## 一、系统概览

```
RepoMind 代码知识库系统
├── 核心功能：代码理解、调用关系分析、智能检索
├── 协议支持：MCP (Model Context Protocol)
├── 语言支持：Python, Java, C/C++, Go, JavaScript/TypeScript
└── 部署方式：本地服务 (HTTP/SSE)
```

## 三、核心组件详细拆解

### 3.1 代码分析流程 (CodeAnalyzer)

```
build_index() 主入口
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤1: 初始化仓库上下文                                   │
│ • 加载Git仓库 (GitPython)                                │
│ • 遍历Git跟踪文件 (遵循.gitignore)                        │
│ • 过滤源代码文件 (FileFilter)                             │
│   - 按扩展名识别语言                                      │
│   - 排除二进制文件、配置文件                              │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤2: 检查增量更新                                       │
│ • 读取缓存的文件元数据 (DiskCache)                        │
│ • 计算文件 hash (SHA1)                                   │
│ • 比对 mtime 和 hash                                     │
│   - 未修改: 跳过                                          │
│   - 已修改: 重新分析                                      │
│   - 新增: 添加到分析列表                                  │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤3: Tree-sitter AST解析                               │
│ • 根据文件语言获取 Parser                                 │
│ • 生成抽象语法树 (AST)                                    │
│ • 使用预定义 Query 提取节点                               │
│   Python: tree-sitter-python                            │
│   Java:   tree-sitter-java                              │
│   Go:     tree-sitter-go                                │
│   C/C++:  tree-sitter-cpp                               │
│   JS/TS:  tree-sitter-javascript/typescript             │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤4: 实体提取 (EntityProcessor)                        │
│ • 函数定义 (function_definition)                         │
│   - 函数名、参数、返回类型                                │
│   - 起止行号                                             │
│   - 完全限定名 (qualified_name)                          │
│ • 类定义 (class_definition)                              │
│   - 类名、继承关系                                        │
│   - 成员变量、成员方法                                    │
│ • 方法定义 (method_definition)                           │
│ • 变量定义 (variable_definition)                         │
│ • 提取文档字符串/注释                                     │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤5: 作用域维护 (ScopeMaintainer)                      │
│ • 构建作用域树                                            │
│   - 全局作用域 → 类作用域 → 函数作用域                   │
│ • 管理符号可见性                                          │
│ • 支持嵌套作用域                                          │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤6: 符号解析 (SymbolResolver)                         │
│ • 记录所有符号定义                                        │
│   {symbol_name → [定义位置列表]}                         │
│ • 建立符号引用关系                                        │
│ • 解析导入语句                                            │
│   - Python: import/from import                          │
│   - Java: import/package                                │
│   - Go: import                                           │
│   - JS/TS: import/require                               │
│ • 处理符号别名                                            │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤7: 调用关系提取 (RelationProcessor)                  │
│ • 识别函数调用表达式                                      │
│   - Python: call_expression                             │
│   - Java: method_invocation                             │
│   - Go: call_expression                                 │
│ • 解析调用目标                                            │
│   - 本地函数                                             │
│   - 类方法                                               │
│   - 外部库函数                                           │
│ • 构建调用图                                             │
│   {caller → [callee1, callee2, ...]}                   │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤8: RepoMap生成                                       │
│ • 生成文件结构映射                                        │
│   {file_path → [definitions]}                           │
│ • 代码分块策略                                            │
│   - 按实体边界分块                                        │
│   - 保留上下文重叠                                        │
│   - chunk_size: 1024, overlap: 200                      │
│ • 为每个分块附加元数据                                    │
│   - rel_fname (相对文件路径)                             │
│   - definitions (定义名称列表)                           │
│   - has_chunk (是否为分块)                               │
└─────────────────────────────────────────────────────────┘
    ↓
返回构建结果 {
    "repo_map_iterator": Iterator[Dict],  # 用于向量化
    "symbol_resolver": SymbolResolver,     # 用于符号查询
    "call_map": Dict[str, List[str]],     # 用于图存储
    "statistics": {
        "total_files": int,
        "analyzed_files": int,
        "skipped_files": int,
        "total_entities": int,
        "total_calls": int
    }
}
```

### 3.2 向量存储流程 (VectorStore)

```
save() 方法
    ↓
┌─────────────────────────────────────────────────────────┐
│ 输入: VectorDataInfo 迭代器                              │
│ • content: 代码内容                                      │
│ • rel_fname: 相对文件路径                                │
│ • definitions: 实体定义列表                              │
│ • has_chunk: 是否为分块                                  │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤1: 文本分块 (Text Splitting)                         │
│ • 使用 RecursiveCharacterTextSplitter                    │
│ • chunk_size = 1024 tokens                              │
│ • chunk_overlap = 200 tokens                            │
│ • 保留代码结构完整性                                      │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤2: 创建 Langchain Document                           │
│ • page_content = 代码内容                                │
│ • metadata = {                                           │
│     "rel_fname": str,                                    │
│     "definitions": List[str],                            │
│     "has_chunk": bool                                    │
│   }                                                      │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤3: 向量嵌入 (Embedding)                              │
│ • 选择嵌入器:                                            │
│   - HuggingFace (默认):                                  │
│     paraphrase-multilingual-MiniLM-L12-v2               │
│   - OpenAI: text-embedding-3-large                      │
│   - BERT: bert-base-uncased                             │
│ • 批量嵌入 (batch_size=500)                              │
│ • 生成向量表示 (384/1536维)                              │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤4: 存储到向量数据库                                   │
│ • FAISS (默认):                                          │
│   - IndexFlatL2 / IndexIVFFlat                          │
│   - 保存到本地文件系统                                    │
│ • Qdrant (可选):                                         │
│   - 支持分布式部署                                        │
│   - 支持过滤查询                                          │
└─────────────────────────────────────────────────────────┘

search() 方法
    ↓
┌─────────────────────────────────────────────────────────┐
│ 输入: query, top_k, definitions, file_paths             │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤1: 查询向量化                                         │
│ • 将自然语言查询转为向量                                  │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤2: 相似度搜索                                         │
│ • 计算余弦相似度 / L2距离                                 │
│ • 返回 Top-K 最相似结果                                   │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤3: 元数据过滤 (MetadataMatcher)                      │
│ • 按 definitions 过滤 (函数名/类名)                       │
│ • 按 file_paths 过滤 (文件路径)                           │
│ • 支持模糊匹配                                            │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 输出: 候选结果列表                                        │
│ [                                                        │
│   {                                                      │
│     "content": str,        # 代码内容                    │
│     "score": float,        # 相似度分数                  │
│     "rel_fname": str,      # 文件路径                    │
│     "definitions": List,   # 定义列表                    │
│     "start_line": int      # 起始行号                    │
│   }                                                      │
│ ]                                                        │
└─────────────────────────────────────────────────────────┘
```

### 3.3 图存储流程 (GraphStore)

```
save() 方法
    ↓
┌─────────────────────────────────────────────────────────┐
│ 输入: nodes_info (NodeInfo), edges_info (EdgeInfo)      │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤1: 添加节点到 NetworkX DiGraph                       │
│ • 节点 ID: 符号名称 (函数名/类名)                         │
│ • 节点属性:                                              │
│   - name: str                                            │
│   - candidates: List[Dict]                              │
│     [                                                    │
│       {                                                  │
│         type: "function",                                │
│         file_path: str,                                  │
│         start_line: int,                                 │
│         signature: str,                                  │
│         comments: str                                    │
│       }                                                  │
│     ]                                                    │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤2: 添加边 (调用关系)                                 │
│ • 边: (source, target)                                   │
│   - source: 调用者函数                                   │
│   - target: 被调用者函数                                 │
│ • 边属性: 调用类型、位置等                                │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ 步骤3: 持久化存储                                         │
│ • 序列化为 GraphML 格式                                  │
│ • 保存到本地文件: callgraph.graphml                      │
│ • 支持增量更新                                            │
└─────────────────────────────────────────────────────────┘

查询方法
    ↓
┌─────────────────────────────────────────────────────────┐
│ get_call_chain_down(function_name, depth)                │
│ • 向下追踪: 该函数调用了哪些函数                          │
│ • 使用 BFS 遍历                                          │
│ • 返回多层调用链                                          │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ get_call_chain_up(function_name, depth)                  │
│ • 向上追溯: 哪些函数调用了该函数                          │
│ • 使用反向图遍历                                          │
│ • 返回调用者链                                            │
└─────────────────────────────────────────────────────────┘
    ↓
┌─────────────────────────────────────────────────────────┐
│ shortest_path(source, target)                            │
│ • 查找两个函数间的最短调用路径                            │

## 四、关键技术点

### 4.1 增量更新机制

```
文件变更检测
├─ 维护文件元数据缓存 (DiskCache)
│  ├─ file_path → { mtime, hash, analysis_result }
│  └─ 持久化到磁盘
├─ 更新判断逻辑
│  ├─ mtime 不变 → 跳过
│  ├─ mtime 变化 + hash 不变 → 跳过 (仅touch)
│  └─ hash 变化 → 重新分析
└─ 优势
   ├─ 大幅减少重复分析
   ├─ 支持大型代码库
   └─ 提升构建速度 (10x+)
```

### 4.2 多语言支持

```
Tree-sitter 生态
├─ 统一的 AST 解析接口
├─ 语言特定的 Query 文件
│  ├─ queries/python/definitions.scm
│  ├─ queries/java/definitions.scm
│  ├─ queries/go/definitions.scm
│  └─ queries/cpp/definitions.scm
├─ EntityProcessor 语言适配
│  ├─ Python: 函数、类、方法
│  ├─ Java: 类、方法、接口
│  ├─ Go: 函数、结构体、接口
│  └─ C/C++: 函数、类、宏
└─ RelationProcessor 调用识别
   ├─ Python: call_expression
   ├─ Java: method_invocation
   ├─ Go: call_expression
   └─ JS/TS: call_expression
```

### 4.3 分布式架构支持

```
组件解耦设计
├─ VectorStore (可独立部署)
│  ├─ 支持 Qdrant 分布式集群
│  └─ 支持 Elasticsearch
├─ GraphStore (可独立部署)
│  ├─ 可迁移到 Neo4j
│  └─ 支持 TigerGraph
├─ MCP Server (可横向扩展)
│  ├─ 无状态设计
│  └─ Session隔离
└─ 缓存层
   ├─ Redis (分布式缓存)
   └─ DiskCache (本地缓存)
```

## 五、性能优化策略

### 5.1 分析阶段优化

```
1. 增量更新
   • 基于文件hash的变更检测

   • 多进程文件解析

   • 限制遍历深度
   • 缓存常用调用链

   • 预过滤减少向量计算
   • 布隆过滤器加速
